# Noun It! — Privacy Policy Update Guide

This document explains how to update `privacy.html` whenever the Termly privacy
policy is regenerated. Follow these steps every time — skipping any of them will
produce a messy diff that is hard to review.

---

## Background

The privacy policy is generated by [Termly](https://termly.io) and exported as
HTML. Termly's export is a single minified block with:

- Inline `<style>` blocks that override fonts and colors with `!important`
- `data-custom-class` attributes on every element
- Inline `style` attributes forcing Arial, specific font sizes, and colors
- `<bdt>` template tags (`block-component`, `question`, `statement-end-if-in-editor`)
- A Termly logo embedded as a base64 SVG
- A duplicate `<h1>` title and "Last updated" line (already rendered by the page wrapper)
- A "Privacy Policy Generator" attribution link at the bottom
- A hidden empty anchor element (`class="privacy123"`)

None of that belongs in the hosted page. The steps below strip it correctly.

---

## Step-by-step update process

### 1. Export from Termly

1. Log in to [app.termly.io](https://app.termly.io)
2. Open the Noun It! privacy policy
3. Click **Copy HTML** (or equivalent export option)

### 2. Save a backup first

Before touching anything, copy the current file to a backup:

```bash
cp noun-it/privacy.html noun-it/privacy-original-backup.html
```

Do not commit this backup — it is for recovery only.

### 3. Paste into the page

Open `noun-it/privacy.html` and paste the Termly HTML between these two markers:

```html
<div class="policy-body">

    <!-- PASTE TERMLY CONTENT HERE -->

</div>
<!-- ══ END POLICY CONTENT ══════════════════════════════════ -->
```

Replace only the content between those markers. Do not touch anything outside them.

### 4. Strip Termly styles and attributes

Run this script from the repo root to remove the inline `<style>` block,
`data-custom-class` attributes, and font/color inline styles — without touching
any content or structure:

```bash
python3 << 'EOF'
import re

with open('noun-it/privacy.html', 'r') as f:
    content = f.read()

before_marker = '<div class="policy-body">'
end_marker = '<!-- ══ END POLICY CONTENT'

before = content[:content.find(before_marker) + len(before_marker)]
after  = content[content.find(end_marker):]
body   = content[content.find(before_marker) + len(before_marker):content.find(end_marker)]

# Remove Termly <style> block
body = re.sub(r'<style[^>]*>.*?</style>', '', body, flags=re.DOTALL)

# Remove data-custom-class attributes
body = re.sub(r'\s*data-custom-class=["\'][^"\']*["\']', '', body)

# Remove inline styles that force Termly fonts/colors
body = re.sub(r'\s*style="[^"]*(?:font-family|font-size|color)[^"]*"', '', body)

with open('noun-it/privacy.html', 'w') as f:
    f.write(before + body + after)

print("Done")
EOF
```

### 5. Format with js-beautify

Termly exports everything as one giant line. Format it so future diffs are
readable. Run from the repo root:

```bash
npx js-beautify --type html --indent-size 2 --wrap-line-length 0 --unformatted "" -r noun-it/privacy.html
```

**Commit this as a formatting-only commit before making any content changes:**

```bash
git add noun-it/privacy.html
git commit -m "Format privacy policy HTML with js-beautify"
```

### 6. Remove Termly-specific elements (content changes)

After the formatting commit, make the following targeted removals. Each is a
precise string replacement — do not use BeautifulSoup or any HTML parser, as
they will reformat the file and pollute the diff.

Manually find and remove each of the following blocks from `privacy.html`:

**a. Termly logo** — a `<span>` with a long base64 SVG background image near
the top of the policy body. It looks like:

```html
<span style="display: block;margin: 0 auto 3.125rem;...background: url(data:image/svg+xml;base64,...)..."></span>
```

Delete the entire `<span>` element.

**b. Duplicate title and date** — Termly includes its own `<h1>` and "Last
updated" line. The page wrapper already renders both. Delete:

```html
<div><strong><span><span>
    <bdt class="block-component"></bdt>
    <bdt class="question">
      <h1>PRIVACY POLICY</h1>
    </bdt>
    <bdt class="statement-end-if-in-editor"></bdt>
  </span></span></strong></div>
<div><span><strong><span><span>Last updated <bdt class="question">...</bdt></span></span></strong></span></div>
```

**c. Termly attribution link** — at the very bottom of the policy body:

```html
<div><span>This Privacy Policy was created using Termly's </span><a href="https://termly.io/products/privacy-policy-generator/" ...>Privacy Policy Generator</a></div>
```

Delete the entire `<div>`.

**d. Hidden empty DSAR anchor** — a hidden `<div>` containing an empty link:

```html
<div style="display: none;"><a class="privacy123" href="https://app.termly.io/dsar/..."></a></div>
```

Delete the entire `<div>`.

**e. Bullet text fix** — Termly renders one bullet as `"Use Noun It!. Trivia
game"` (with a stray period). Fix it to read:

```
Use the Noun It! trivia game
```

**Commit these content changes separately:**

```bash
git add noun-it/privacy.html
git commit -m "Remove Termly artifacts and fix content in privacy policy"
```

---

## What to leave alone

- All `<bdt>` tags — they are harmless in the browser and touching them risks
  losing content
- All internal anchor links (`#toc`, `#infocollect`, etc.) — verified working
- All `mailto:support@pixerova.com` links — appear 5 times, all correct
- All DSAR links to `app.termly.io/dsar/...` — these point to Termly's hosted
  data subject access request form and should stay as long as the Termly account
  is active

---

## Update the "Last updated" date

After pasting new Termly content, update the date in the page wrapper
(not inside the Termly block):

```html
<p class="policy-meta">
  <span ...>App: Noun It! &nbsp;·&nbsp; Last updated:</span>
  <!-- UPDATE THIS DATE when the policy changes -->
  April 2026   ← update this
</p>
```
